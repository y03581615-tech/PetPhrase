# PetPhrase 设计文档

日期:2026-07-04
状态:待用户确认

## 1. 产品定义

Windows 桌面常驻小工具:一只 petdex 格式的桌宠静坐屏幕角落,点击弹出常用语面板,点击常用语即复制到剪贴板。轻量、美观、两次点击完成一次复制。

**目标用户**:高频重复输入文字的 Windows 用户(客服/运营/开发者),以及桌宠爱好者。

**成功标准**:
- 点击宠物到面板出现 < 200ms;点击短语到剪贴板写入 < 100ms
- 常驻内存 < 40MB;安装包 < 10MB;冷启动 < 1s
- 不看说明书即会用(宠→语→贴 两次点击心智)

**范围声明**:本 MVP 即产品全部范围,无既定迭代路线。以下明确不做:自动粘贴、全局快捷键、剪贴板历史、宠物游走行为、在线宠物画廊、富文本/变量短语、云同步、多宠同屏。

## 2. 核心交互流

```
桌宠(静坐 idle 动画,可拖拽,位置记忆)
  │ 单击
  ▼
常用语面板(贴宠弹出,失焦即隐)
  │ 单击某条短语
  ▼
写入剪贴板 → 短语条目闪一下「已复制 ✓」→ 面板自动收起 → 宠物播 wave 动画
```

托盘图标:显示/隐藏宠物、打开设置、退出。

## 3. UI/UX 决议(浏览器协作确认)

### 3.1 面板结构 —— B 案:Tab + 自适应混排
- 尺寸 300×400px(逻辑像素),贴宠弹出,自动避让屏幕边缘
- 顶部:搜索框 + 设置齿轮
- 分组条:胶囊 Tab,点击切组;记住上次所在分组
- 内容区按短语长度自适应:
  - 短句(≤10 字符)排成**气泡流**,一行 2-3 个
  - 长句独占卡片,**钳制 2 行** + 省略号
- 一屏可见约 10-14 条,超出纵向滚动,不设上限

### 3.2 分组溢出 —— C 案:滚动 Tab + ⊞ 全组宫格
- Tab 条可横向滚动(滚轮映射横滚),右缘渐隐提示
- 右端常驻「⊞」按钮:展开全组宫格,所有分组带图标平铺、位置固定,点选切换并收起
- 分组 ≤4 个时「⊞」自动隐藏
- 分组名自定义 + 可选图标,设置里拖拽排序;不限制建组数,UI 按 ≤8 组优化
- 搜索时无视分组、全库过滤,结果标注来源分组

### 3.3 长文本预览 —— A 案:侧边悬浮全文卡
- 仅被截断的条目,悬停 400ms 后在面板**外侧**浮出深色全文卡
- 列表零抖动、零遮挡;贴屏幕边时浮层自动换侧
- 移开或 Esc 关闭;单击仍是复制

### 3.4 视觉风格 —— 默认亚克力玻璃(C),暖橘奶油(A)为候选主题
- **主题走 design token(CSS 变量)架构**,皮肤可切换,新增主题零重构
- 默认主题「亚克力」:半透明磨砂透出壁纸,Win11 原生感
  - 真·亚克力经 `window-vibrancy` crate(`apply_acrylic`/`apply_mica`),非 CSS backdrop-filter(后者无法模糊窗外桌面)
  - 可读性兜底:白底不透明度设下限;系统「透明效果」关闭或效果不可用时自动退化为实底浅色主题
- 候选主题「暖橘奶油」(claymorphism):后续以第二主题加入,设置里切换
- 对比度一律按 WCAG 4.5:1 校验;悬停全文浮层保持深色底
- 图标用 SVG(Lucide),不用 emoji 作功能图标;动效 150-300ms、只动 transform/opacity、尊重 prefers-reduced-motion

## 4. 技术栈

**Tauri v2**(Rust 后端 + WebView2)+ 前端 Vite + TypeScript(不上重框架,原生 DOM/轻量响应式即可)。

选型对比结论:Electron 违背轻量硬指标;WPF 美化成本高且用户不熟 C#;纯 Rust GUI(egui/slint)做美观面板成本翻数倍。Tauri 官方插件覆盖全部需求:`clipboard-manager`、`autostart`、`tray-icon`、`store`、`single-instance`。已知代价:依赖 WebView2(Win10/11 自带,安装器引导);Rust 编译慢(不影响前端热更新)。

## 5. 架构

```
┌─ Tauri App(单进程,single-instance)────────────┐
│  Rust 核心                                      │
│  ├─ PetLoader:扫描宠物目录,解析 pet.json,      │
│  │   校验雪碧图(8行×9列,192×208/帧),           │
│  │   asset 协议供图;格式解析独立模块隔离          │
│  ├─ PhraseStore:phrases.json 读写(serde),      │
│  │   %APPDATA%/PetPhrase/,原子写 + 启动时备份    │
│  ├─ Settings:settings.json(当前宠/主题/         │
│  │   宠物位置/上次分组/自启开关)                  │
│  └─ 插件:clipboard / tray / autostart / store   │
│                 │ IPC(commands + events)        │
│  WebView2 前端(3 窗口)                          │
│  ├─ ① 宠物窗:透明+置顶+无边框+不进任务栏,        │
│  │    雪碧图 CSS steps() 动画,拖拽,单击开面板    │
│  ├─ ② 面板窗:300×400 无边框,常驻隐藏而非现建     │
│  │    (保 200ms 弹出),失焦即隐,acrylic 背景     │
│  └─ ③ 设置窗:常规窗口,短语 CRUD/分组管理/        │
│       换宠/主题/导入导出 JSON/开机自启            │
└──────────────────────────────────────────────────┘
数据流:设置窗改数据 → Rust 写盘 → event 广播 → 面板/宠物窗刷新
```

**数据文件**(均在 `%APPDATA%/PetPhrase/`):
- `phrases.json`:`{ groups: [{ id, name, icon, order, phrases: [{ id, text, order }] }] }`
- `settings.json`:当前宠物 id、主题、宠物窗位置、上次分组、自启开关
- 单进程 + 原子写,不引入 SQLite

**宠物目录扫描**(三处,按序合并):应用内置目录 → `~/.codex/pets/` → 用户自定义目录。严格支持 petdex 格式(`pet.json` + `spritesheet.webp/png`)作为唯一格式,内置宠同格式,用户换形象零额外成本。

## 6. 错误处理

- 宠物包校验失败(缺文件/尺寸不符):设置里该宠标灰 + 具体原因,不崩溃,回退默认宠
- `phrases.json` 损坏:启动时自动从上次备份恢复,托盘气泡告知
- 剪贴板写入失败(被占用):条目上显示「复制失败,重试」,不静默
- WebView2 缺失:安装器自动引导安装
- 系统透明效果不可用:主题自动退化实底,无感知降级

## 7. 测试

- 单元:PhraseStore 读写/备份恢复、PetLoader 校验(Rust `#[test]`);前端搜索过滤、长度分类(Vitest)
- 集成:IPC commands 端到端(tauri 测试工具)
- 视觉/手动清单:三窗口 DPI 缩放(100%/150%/200%)、多显示器、贴边浮层换侧、透明效果开关降级、拖拽与单击不误触
- 性能验收:按第 1 节指标实测内存/启动/响应

## 8. 风险与债

| 风险 | 级别 | 对策 |
|------|------|------|
| 内置宠素材版权(petdex 素材归投稿者自定 license) | 高 | 发布前核实;不可再分发则自绘默认宠,petdex 宠一律用户自装 |
| 透明窗口点击穿透/拖拽冲突/DPI | 中 | 开发首周先做技术验证 spike,再铺功能 |
| acrylic 在部分机器掉帧/发灰 | 中 | 面板静止显示影响有限;实测不佳则默认主题切实底,acrylic 变可选 |
| petdex 格式无正式规范、可能演进 | 中 | PetLoader 单模块隔离,校验失败友好报错 |
| WebView2 空闲内存爬升 | 中 | 面板隐藏时释放非必要 DOM;验收实测 |

**主动接受的债**:行为状态机只留接口不实现(宠物窗动画模块独立);暖橘主题延后;主色跟随宠物自动提取延后。
