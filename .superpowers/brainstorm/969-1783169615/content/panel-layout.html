A · 输入法式 Tab + 统一行列表(你的初想)

顶部搜索 + 分组胶囊 Tab,点 Tab 切组。每条一行卡片:短句单行,长句钳制 2 行 + 悬停看全文。一屏约 7-9 条,超出滚动。结构最直白,和手机输入法心智一致。

B · Tab + 自适应混排(短句成气泡流,长句成卡片)

同样的 Tab 结构,但内容区按长度自适应:短句(≤10字)自动排成气泡流,2-3 个一行;长句独占卡片钳 2 行。短句密度翻倍,一屏可见 10-14 条,视觉更活泼、更「宠物感」。

C · 无 Tab 全量滚动 + 粘性分组头

不切组,所有分组连续滚动,组名吸顶。「一屏看到所有」的直觉最强,搜索时天然跨组。但组多/条多时滚动成本高,分组的「空间记忆」弱于 Tab。回答你的问题:点击展开全部并非最优——超过 ~30 条后找东西靠滚,不如 Tab 定位快。

我的推荐:A 和 B 同架构,B 是 A 的密度升级

你的输入法直觉是对的:Tab 切组 = 一次点击直达,肌肉记忆强,面板恒定小巧。C 只适合短语很少的用户。
我推荐 B:架构与 A 完全相同(后续可做成开关),但短句自动成气泡流——常用语大多是短句,密度收益明显,且气泡更配桌宠的可爱气质。
尺寸结论(下屏细聊):300×400px,约屏高 1/3,不遮内容;一屏 7-14 条可见,滚动上限不设。分组过多的处理(Tab 溢出)也在后面单独一屏讨论。
